# Cobblemon 幸运方块数据包 - 使用说明

## 📦 数据包信息

- **名称**: Cobblemon Lucky Block
- **版本**: 1.0.0
- **适用版本**: Minecraft 1.21.1 + Cobblemon
- **作者**: 自动生成

## ✨ 功能特性

### 1. 随机宝可梦生成
- 打破幸运方块时生成一只等级 50 的随机宝可梦
- 使用 Cobblemon 的 `/pokespawn random` 命令

### 2. 闪光系统
- **10% 闪光几率** - 每 10 次就有 1 次机会生成闪光宝可梦
- 闪光宝可梦有特殊的粒子效果和声音

### 3. 视觉效果
- 快乐村民粒子效果
- 发光粒子效果
- 附魔击中粒子效果
- 经验球拾取声音
- 闪光时的末影龙成长声音

### 4. 聊天消息
- 生成普通宝可梦：金色加粗消息
- 生成闪光宝可梦：彩色渐变消息，带闪光标识

## 🚀 安装步骤

### 步骤 1: 准备环境
确保你已经安装了:
- Minecraft Java Edition 1.21.1
- Fabric 或 NeoForge 加载器
- Cobblemon 模组 (最新版本)

### 步骤 2: 安装数据包

1. 找到你的 Minecraft 世界存档文件夹
   - 默认位置：`%appdata%\.minecraft\saves\你的世界名\datapacks`
   - 或者在服务器中：`服务器根目录/world/datapacks`

2. 将 `lucky_block_datapack` 整个文件夹复制到 `datapacks` 文件夹中

3. 确保文件夹结构如下:
   ```
   datapacks/
   └── lucky_block_datapack/
       ├── pack.mcmeta
       ├── README.md
       └── data/
           ├── lucky_block/
           │   ├── advancement/
           │   ├── function/
           │   └── tags/
           └── minecraft/
               └── tags/
   ```

### 步骤 3: 加载数据包

**单人游戏:**
1. 进入你的世界
2. 按 `T` 打开聊天栏
3. 输入 `/reload` 并回车
4. 看到加载成功消息

**服务器:**
1. 在服务器控制台输入 `/reload`
2. 或者重启服务器

## 🎮 使用方法

### 获得幸运方块

在游戏中使用以下命令:
```
/function lucky_block:give_lucky_block
```

这会给你一个特殊的金块 (幸运方块)，带有:
- 金色自定义名称 "幸运方块"
- 描述性 Lore
- LuckyBlock NBT 标签

### 使用幸运方块

1. **放置方块**: 将幸运方块放置在你想要的位置
2. **打破方块**: 用任何工具 (或徒手) 打破幸运方块
3. **享受惊喜**: 一只随机的等级 50 宝可梦会出现在你面前!

### 其他有用命令

- `/datapack list` - 查看已加载的数据包列表
- `/function lucky_block:init` - 重新初始化数据包
- `/advancement revoke @s lucky_block:break_lucky_block` - 重置进度 (用于测试)

## ⚙️ 自定义配置

### 修改宝可梦等级

编辑文件：`data/lucky_block/function/spawn_normal.mcfunction` 和 `spawn_shiny.mcfunction`

找到这一行:
```
execute as @p at @s positioned ~ ~1 ~ run pokespawn random level=50
```

将 `level=50` 改为你想要的等级 (1-100)

### 修改闪光几率

编辑文件：`data/lucky_block/function/on_break.mcfunction`

找到这一行:
```
scoreboard players random lucky_shiny 0 9
```

- `0 9` = 10% 几率 (0-9 共 10 个数，0 为闪光)
- `0 19` = 5% 几率
- `0 4` = 20% 几率
- `0 99` = 1% 几率

### 修改粒子效果

编辑文件：`data/lucky_block/function/on_break.mcfunction`

可用的粒子类型:
- `minecraft:happy_villager` - 快乐村民粒子
- `minecraft:glow` - 发光粒子
- `minecraft:enchanted_hit` - 附魔击中粒子
- `minecraft:crit` - 暴击粒子
- `minecraft:spell` - 魔法粒子
- `minecraft:portal` - 传送门粒子

### 修改生成消息

编辑文件：`data/lucky_block/function/spawn_normal.mcfunction` 和 `spawn_shiny.mcfunction`

使用 JSON 格式自定义聊天消息

## 🔧 故障排除

### 数据包无法加载

**问题**: 使用 `/reload` 后没有看到数据包加载消息

**解决方案**:
1. 检查文件夹结构是否正确
2. 确保 `pack.mcmeta` 文件存在
3. 使用 `/datapack list` 查看是否在列表中
4. 检查是否有语法错误

### 宝可梦不生成

**问题**: 打破金块后没有宝可梦出现

**解决方案**:
1. 确保 Cobblemon 模组已正确安装
2. 测试 `/pokespawn random level=50` 命令是否可用
3. 检查是否有足够的空间生成宝可梦
4. 查看聊天栏是否有错误消息

### 所有金块都触发效果

**问题**: 打破任何金块都会生成宝可梦

**解决方案**:
这是当前版本的特性。如果你想只对特定金块生效:
1. 使用给予函数获得的金块才是幸运方块
2. 或者手动给金块添加 NBT 标签

### 闪光宝可梦不出现

**问题**: 从来没有见过闪光宝可梦

**解决方案**:
1. 闪光几率是 10%，可能需要多次尝试
2. 检查 `on_break.mcfunction` 中的计分板设置
3. 确保 Cobblemon 支持闪光宝可梦

## 📝 技术细节

### 数据包结构

```
lucky_block_datapack/
├── pack.mcmeta                 # 数据包元数据
├── README.md                   # 说明文档
└── data/
    ├── lucky_block/
    │   ├── advancement/
    │   │   └── break_lucky_block.json  # 金块破坏进度
    │   ├── function/
    │   │   ├── init.mcfunction         # 初始化函数
    │   │   ├── tick.mcfunction         # 每 tick 函数
    │   │   ├── on_break.mcfunction     # 金块破坏主函数
    │   │   ├── spawn_normal.mcfunction # 生成普通宝可梦
    │   │   ├── spawn_shiny.mcfunction  # 生成闪光宝可梦
    │   │   └── give_lucky_block.mcfunction # 给予幸运方块
    │   └── tags/
    └── minecraft/
        └── tags/
            └── function/
                ├── load.json   # 加载标签
                └── tick.json   # Tick 标签
```

### 工作原理

1. **进度触发**: 使用 Minecraft 进度系统检测金块破坏
2. **函数调用**: 触发 `on_break` 函数
3. **效果播放**: 生成粒子和声音效果
4. **随机判定**: 使用计分板随机数决定闪光
5. **宝可梦生成**: 调用 Cobblemon 的 `/pokespawn` 命令

### 计分板系统

- **lucky_shiny**: 用于闪光判定的临时计分板
- 范围：0-9 (10% 几率)
- 每次破坏后自动重置
