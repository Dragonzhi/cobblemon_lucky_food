# 🎯 幸运方块 - 关键修复说明

## ❌ 之前的问题

`using_item` 进度**只会检测可以消耗的物品**！

普通的鸡蛋不能被"使用"（右键），所以进度永远不会触发。

## ✅ 解决方案

给物品添加 `consumable` 组件，使其可以被右键使用：

```json
consumable={
  consume_seconds: 0,
  animation: "none"
}
```

### 参数说明

- **consume_seconds: 0** - 立即消耗（0 秒）
- **animation: "none"** - 不显示使用动画

这样物品就可以被右键使用，但不会真正被消耗（除非我们主动清除它）

## 🔧 技术细节

### 为什么需要 consumable 组件？

在 Minecraft 1.20.5+ 中，`using_item` 进度只会在以下情况触发：
1. 玩家使用食物
2. 玩家使用药水
3. 玩家使用带有 `consumable` 组件的物品

**关键点：** 物品必须能被"使用"才会触发进度！

### consumable 组件的作用

```json
consumable={
  consume_seconds: 0,      // 使用所需时间（tick）
  animation: "none"        // 使用动画：none, eat, drink
}
```

- **consume_seconds: 0** - 立即完成使用
- **animation: "none"** - 不显示手臂动画

这样物品可以触发使用事件，但不会真正被消耗。

### 物品消耗

我们在 `on_use.mcfunction` 中手动清除物品：

```mcfunction
# 消耗幸运蛋
clear @p minecraft:egg[custom_data={lucky_block:1b}] 1
```

## 🧪 测试验证

### 测试步骤

1. **给予物品**
   ```
   /function lucky_block:give_lucky_block
   ```

2. **检查 NBT**
   ```
   /data get entity @s SelectedItem
   ```
   应该看到 `consumable` 组件

3. **右键使用**
   对着空气右键

4. **验证效果**
   应该看到粒子、声音、宝可梦

### 常见问题

**Q: 物品被消耗但不生成宝可梦？**
```
A: 检查进度是否正确触发
   /advancement show @s lucky_block:use_lucky_block
```

**Q: 右键没有效果？**
```
A: 确保物品有 consumable 组件
   /data get entity @s SelectedItem
```

**Q: 如何修改使用动画？**
```
A: 修改 give_lucky_block.mcfunction 中的 animation:
   - "none" - 无动画
   - "eat" - 吃东西动画
   - "drink" - 喝东西动画
```
