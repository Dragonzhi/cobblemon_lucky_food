# 🔧 幸运方块数据包 - 实现方式变更说明

## 📋 变更原因

**问题：** 之前使用物品 NBT/组件的方式，当金块放置后数据会丢失，导致幸运方块变成普通金块。

**解决方案：** 使用 Minecraft 的战利品表系统覆盖所有金块的掉落表。

## 🎯 新的实现方式

### 工作原理

1. **战利品表覆盖** - 数据包覆盖了 `minecraft:gold_block` 的战利品表
2. **触发机制** - 当任何金块被破坏时，会触发 `lucky_block:on_break` 函数
3. **效果执行** - 函数执行粒子效果、声音和宝可梦生成

### 文件结构

```
data/minecraft/loot_tables/blocks/gold_block.json
  ↓
触发 lucky_block:on_break.mcfunction
  ↓
执行效果并生成宝可梦
```

## ✅ 优点

1. **简单可靠** - 不依赖物品 NBT，所有金块都有效
2. **易于理解** - 玩家知道金块就是幸运方块
3. **无需特殊处理** - 直接给予普通金块即可

## ⚠️ 注意事项

### 重要提示

**现在所有金块都是幸运方块！**

这意味着：
- ✅ 给予的金块可以使用
- ✅ 放置的金块可以使用
- ✅ 任何来源的金块都可以触发效果
- ⚠️ 破坏任何金块都会生成宝可梦（包括你不想触发的时候）

### 使用建议

1. **谨慎放置金块** - 确保你想让它生成宝可梦
2. **创造模式注意** - 在创造模式下破坏金块也会触发
3. **自动化农场** - 如果有金块农场，需要小心处理

## 🎮 使用方法

### 获得幸运方块
```
/function lucky_block:give_lucky_block
```

### 使用步骤
1. 放置金块
2. 打破金块
3. 享受惊喜！✨

## 🔍 技术细节

### 战利品表格式
```json
{
  "type": "minecraft:block",
  "pools": [
    {
      "rolls": 1,
      "entries": [
        {
          "type": "minecraft:item",
          "name": "minecraft:gold_block"
        }
      ]
    }
  ],
  "functions": [
    {
      "function": "minecraft:run_function",
      "name": "lucky_block:on_break"
    }
  ]
}
```

### 执行流程
```
玩家破坏金块
    ↓
战利品表触发
    ↓
执行 on_break 函数
    ↓
1. 粒子效果
2. 声音效果
3. 随机闪光判定
4. 生成宝可梦
5. 返回金块物品
```

## 🐛 故障排除

### 如果还是没有效果

1. **检查战利品表是否加载：**
   ```
   /datapack list
   ```

2. **手动测试函数：**
   ```
   /function lucky_block:on_break
   ```

3. **检查金块是否正确：**
   - 确保是 `minecraft:gold_block`
   - 不是其他模组的金块

### 如果所有金块都触发（这是特性！）

这是新版本的正常行为。如果只想在特定情况下触发：
- 在创造模式下小心破坏金块
- 或者暂时禁用数据包
