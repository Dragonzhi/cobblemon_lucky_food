# 🎉 幸运方块数据包 - 物品触发版

## ✅ 最终最终方案！

### 实现方式
使用 **鸡蛋道具 + 自定义数据 + 使用物品进度** 来触发

## 🎯 工作原理

### 1. 获得幸运方块道具
```
/function lucky_block:give_lucky_block
```
获得带有 `custom_data={lucky_block:1b}` 的鸡蛋

### 2. 右键使用
玩家右键使用幸运蛋时触发

### 3. 进度检测
`using_item` 进度检测带有标记的鸡蛋被使用

### 4. 生成宝可梦
执行 `on_use` 函数，生成宝可梦

## 🔍 技术细节

### 进度触发器
```json
{
  "criteria": {
    "use_lucky_block": {
      "trigger": "minecraft:using_item",
      "conditions": {
        "item": {
          "items": ["minecraft:egg"],
          "predicates": {
            "custom_data": "{lucky_block:1b}"
          }
        }
      }
    }
  }
}
```

### Tick 重置
```mcfunction
execute as @a[nbt={Advancements:{lucky_block:use_lucky_block:true}}] 
run advancement revoke @s lucky_block:use_lucky_block
```

### 使用函数
```mcfunction
# 粒子效果
particle minecraft:happy_villager ~ ~1 ~ 0.5 0.5 0.5 0.1 50
particle minecraft:glow ~ ~1 ~ 0.5 0.5 0.5 0.1 30
particle minecraft:enchanted_hit ~ ~1 ~ 0.5 0.5 0.5 0.1 20

# 声音
playsound minecraft:entity.experience_orb.pickup master @a ~ ~ ~ 1 1

# 随机闪光
scoreboard players random lucky_shiny 0 9
execute if score lucky_shiny matches 0 run function lucky_block:spawn_shiny
execute if score lucky_shiny matches 1..9 run function lucky_block:spawn_normal
```

## 🐛 故障排除

### 如果不生成宝可梦

1. **检查进度是否加载：**
   ```
   /advancement show @s lucky_block:use_lucky_block
   ```

2. **手动测试函数：**
   ```
   /function lucky_block:on_use
   ```

3. **检查物品 NBT：**
   ```
   /data get entity @s SelectedItem
   ```
   应该看到 `custom_data:{lucky_block:1b}`

### 如果只能触发一次

Tick 函数应该自动重置进度。检查：
```
/datapack list
```
确保数据包正常加载。
