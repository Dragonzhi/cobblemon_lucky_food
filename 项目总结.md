# 📦 Cobblemon 幸运方块数据包 - 项目总结

## 项目概述

基于 Cobblemon 模组实现的 Minecraft 数据包，添加了可以生成随机宝可梦的幸运方块。

### 核心功能
✅ 打破金块生成等级 50 的随机宝可梦  
✅ 10% 几率生成闪光宝可梦  
✅ 华丽的粒子效果和声音  
✅ 精美的聊天消息提示

## 技术实现

### 1. 进度系统
使用 Minecraft 原生进度系统检测金块破坏：
```json
{
  "criteria": {
    "break_gold_block": {
      "trigger": "minecraft:player_destroyed_block",
      "conditions": {
        "block": {
          "blocks": ["minecraft:gold_block"]
        }
      }
    }
  },
  "rewards": {
    "function": "lucky_block:on_break"
  }
}
```

### 2. 随机闪光系统
使用计分板实现 10% 闪光几率：
```mcfunction
scoreboard players random lucky_shiny 0 9
execute if score lucky_shiny matches 0 run function lucky_block:spawn_shiny
execute if score lucky_shiny matches 1..9 run function lucky_block:spawn_normal
```

### 3. 粒子效果组合
```mcfunction
# 基础效果
particle minecraft:happy_villager ~ ~1 ~ 0.5 0.5 0.5 0.1 50
particle minecraft:glow ~ ~1 ~ 0.5 0.5 0.5 0.1 30
particle minecraft:enchanted_hit ~ ~1 ~ 0.5 0.5 0.5 0.1 20

# 闪光额外效果
particle minecraft:glow ~ ~2 ~ 1 1 1 0.1 100
```

### 4. Cobblemon 命令集成
```mcfunction
# 普通宝可梦
execute as @p at @s positioned ~ ~1 ~ run pokespawn random level=50

# 闪光宝可梦
execute as @p at @s positioned ~ ~1 ~ run pokespawn random level=50 shiny
```

## 自定义选项

### 可配置参数

| 参数 | 默认值 | 修改文件 | 说明 |
|------|--------|----------|------|
| 宝可梦等级 | 50 | spawn_normal.mcfunction, spawn_shiny.mcfunction | 1-100 |
| 闪光几率 | 10% | on_break.mcfunction | 修改 random 范围 |
| 生成位置 | 玩家上方 1 格 | 所有 spawn 函数 | 修改 positioned 坐标 |
| 粒子类型 | 多种组合 | on_break.mcfunction | Minecraft 粒子 ID |
| 声音效果 | experience_orb | on_break.mcfunction | Minecraft 声音 ID |
| 聊天消息 | 金色加粗 | spawn_normal/shiny.mcfunction | JSON 格式 |

### 修改示例

**降低闪光几率到 5%:**
```mcfunction
# on_break.mcfunction
scoreboard players random lucky_shiny 0 19  # 0-19 共 20 个数，0 为闪光 = 5%
```

**修改宝可梦等级到 100:**
```mcfunction
# spawn_normal.mcfunction 和 spawn_shiny.mcfunction
execute as @p at @s positioned ~ ~1 ~ run pokespawn random level=100
```

## 性能考虑

### 优化点
1. **进度系统** - 使用 Minecraft 原生系统，性能开销最小
2. **粒子数量** - 控制在合理范围 (50-100 个)
3. **声音范围** - 限制在 15 格以内
4. **计分板** - 使用后立即重置，避免内存泄漏

### 潜在问题
- ⚠️ 大量玩家同时打破幸运方块可能导致短暂卡顿
- ⚠️ 闪光效果粒子较多，低配置电脑可能掉帧

## 扩展建议

### 未来可能添加的功能

1. **幸运方块等级系统**
   - 不同等级的幸运方块生成不同等级的宝可梦
   - 使用不同的方块类型 (金块、钻石块、绿宝石块等)

2. **群系相关生成**
   - 根据所在群系生成对应类型的宝可梦
   - 使用 execute in 命令检测群系

3. **团队/阵营系统**
   - 记录打破幸运方块的玩家
   - 生成的宝可梦属于打破者

4. **冷却系统**
   - 防止短时间内大量生成
   - 使用 scoreboard 计时

5. **统计系统**
   - 记录打破的幸运方块数量
   - 记录生成的闪光宝可梦数量
   - 使用 advancement 或 scoreboard

6. **自定义掉落表**
   - 除了宝可梦，还可以生成道具
   - 使用 loot_table 系统

## 已知限制

1. **金块覆盖** - 当前版本所有金块都会触发效果
   - 解决方案：使用给予函数获得的 NBT 金块

2. **宝可梦选择** - 使用完全随机，无法指定特定宝可梦
   - 解决方案：修改 Cobblemon 配置或使用特定命令

3. **生成位置** - 固定在玩家上方 1 格
   - 解决方案：可以修改为随机位置或指定方向
